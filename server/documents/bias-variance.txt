Title: Diagnostics for Machine Learning: Bias vs. Variance
Speaker/Author: Andrew Ng

When developing a machine learning model, one of the most common issues is that your model might perform poorly on new data. This is typically due to either "high bias" (underfitting) or "high variance" (overfitting). Understanding which of these two is the problem is the most important step in diagnostic testing for machine learning.

Let's define these terms:
1. High Bias (Underfitting): The model is too simple to capture the underlying pattern of the data. For example, fitting a straight line to data that is quadratic. A high bias model has high error on both the training set and the test set.
2. High Variance (Overfitting): The model is too complex and fits the noise in the training set rather than the underlying pattern. For example, fitting a high-degree polynomial (e.g., 5th degree) to a small dataset. A high variance model has very low training error but very high test error (generalization gap).

How do you diagnose this?
You should look at the Training Error and the Validation (Dev) Error:
- If Training Error is high, and Validation Error is high, you have a Bias problem.
- If Training Error is low, and Validation Error is high, you have a Variance problem.
- If Training Error is high, and Validation Error is even higher, you may have both.
- If Training Error is low, and Validation Error is low, congratulations! Your model is performing well.

How do you fix these issues?
If you have High Bias (underfitting):
- Try a larger neural network (more layers or more hidden units).
- Train the model longer or use better optimization algorithms (like Adam).
- Try different neural network architectures.
- Decrease regularization (e.g., reduce lambda).

If you have High Variance (overfitting):
- Get more training data. This is often the most effective fix, if available.
- Try regularization. L2 regularization (weight decay) penalizes large weights, which prevents the network from fitting noise too closely.
- Try dropout (randomly disabling neurons during training).
- Reduce the number of features or simplify the network architecture.

I often see engineers try to get more data to fix an underfitting model, or build a larger network to fix an overfitting model. This is a waste of time! Diagnosing bias vs. variance first will save you weeks of work.
